## 6.1 一般消費

### 現金消費

#### 記帳方式
```
交易類型：支出
支付給：E-餐飲費
從科目：A-現金
金額：50
明細：午餐費
```

#### 帳務影響
- 現金減少 50 元
- 餐飲費支出增加 50 元

### 銀行轉帳消費

#### 記帳方式
```
交易類型：支出
支付給：E-網路購物
從科目：A-銀行帳戶
金額：1500
明細：網路購物
```

#### 帳務影響
- 銀行帳戶減少 1500 元
- 購物支出增加 1500 元

### 其他消費範例

#### 交通費
```
支付給：E-交通費
從科目：A-現金
金額：30
明細：捷運
```

#### 娛樂費
```
支付給：E-娛樂費
從科目：A-信用卡
金額：500
明細：電影票
```

## 6.2 信用卡消費及繳款

### 核心概念
- 信用卡視為負債科目（L-信用卡）
- 刷卡時從信用卡支出
- 繳款時從銀行轉帳到信用卡

### 1. 刷卡消費記錄

#### 記帳方式
```
交易類型：支出
支付給：E-網路購物（或其他消費科目）
從科目：L-信用卡
金額：1500
明細：購買電腦配件
```

#### 帳務影響
- 信用卡負債增加 1500 元
- 對應支出增加 1500 元

### 2. 信用卡利息支付

#### 記帳方式
```
交易類型：支出
支付給：E-利息支出
從科目：L-信用卡
金額：200
明細：信用卡利息
```

#### 說明
- 信用卡產生的利息應單獨記錄
- 列為利息支出科目
- 增加信用卡負債金額

### 3. 信用卡繳款

#### 記帳方式
```
交易類型：轉帳
轉換到：L-信用卡
從科目：A-銀行帳戶
金額：5000
明細：繳信用卡費
```

#### 帳務影響
- 銀行帳戶減少 5000 元
- 信用卡負債減少 5000 元
- 總資產不變（資產減少，負債也減少）

### 4. 刷退處理

#### 記帳方式
```
交易類型：支出
支付給：E-網路購物
從科目：L-信用卡
金額：-1500（負數）
明細：退貨
```

#### 說明
- 若商品退回或消費取消
- 採用相同方式記帳，但將金額改為**負數**
- 負數支出等於減少負債

### 業務邏輯

#### 完整流程範例
```
1/5  刷卡消費：L-信用卡 → E-購物 1500 元
1/10 刷卡消費：L-信用卡 → E-餐飲費 800 元
1/15 產生利息：L-信用卡 → E-利息 50 元
1/20 繳款：A-銀行帳戶 → L-信用卡 2350 元

信用卡餘額變化：
0 → -1500 → -2300 → -2350 → 0
```

### 優點
- 清晰區分消費與支付流程
- 準確追蹤信用卡債務
- 利息單獨記錄便於分析

## 6.4 借錢及還錢

### 借錢給別人

#### 建立科目
- 科目名稱：A-應收帳款
- 科目類型：資產

#### 借出時記帳
```
交易類型：轉帳
轉換到：A-應收帳款
從科目：A-現金
金額：500
明細：借 500 元給張三
```

#### 對方還錢時記帳
```
交易類型：轉帳
轉換到：A-現金
從科目：A-應收帳款
金額：500
明細：張三還 500 元
```

#### 帳務影響
```
借出時：
- 現金減少 500 元
- 應收帳款增加 500 元
- 總資產不變（資產型態轉換）

收回時：
- 應收帳款減少 500 元
- 現金增加 500 元
- 總資產不變
```

### 跟別人借錢

#### 建立科目
- 科目名稱：L-應付帳款
- 科目類型：負債

#### 借入時記帳
```
交易類型：轉帳
轉換到：A-現金
從科目：L-應付帳款
金額：500
明細：跟李四借 500 元
```

#### 還款時記帳
```
交易類型：轉帳
轉換到：L-應付帳款
從科目：A-現金
金額：500
明細：還李四 500 元
```

#### 帳務影響
```
借入時：
- 現金增加 500 元
- 應付帳款（負債）增加 500 元
- 淨資產不變

還款時：
- 現金減少 500 元
- 應付帳款（負債）減少 500 元
- 淨資產不變
```

### 利息處理

#### 收取利息（借錢給別人）
```
進帳從：I-利息收入
到科目：A-現金
金額：50
明細：張三還款利息
```

#### 支付利息（跟別人借錢）
```
支付給：E-利息支出
從科目：A-現金
金額：50
明細：支付李四借款利息
```

## 6.6 遺失錢及撿到錢

### 遺失金錢的記錄

#### 記帳方式
```
交易類型：支出
支付給：E-其他支出
從科目：A-現金
金額：100
明細：遺失
```

#### 帳務影響
- 現金減少 100 元
- 其他支出增加 100 元
- 總資產減少 100 元

### 撿到錢的記錄

#### 方法一：記為其他收入
```
交易類型：收入
進帳從：I-其他收入
到科目：A-現金
金額：100
明細：撿到錢
```

#### 方法二：使用 Equity（MyAB 專用）
```
交易類型：轉帳
轉換到：A-現金
從科目：Equity
金額：100
明細：撿到錢
```

#### 兩種方法的差異
- **方法一**：計入收入，會影響損益表
- **方法二**：不計入收入，僅調整資產

- A-互助會帳戶（資產科目）

```
明細：1月份會錢

```

```
交易類型：收入
進帳從：I-利息收入
```

## 6.8 儲值卡

### 核心概念
- 儲值卡視為資產科目
- 儲值時增加資產
- 消費時減少資產
- 贈送金額視為收入

### 狀況 A：含贈送金額的儲值卡

#### 情境
- 購買餐廳儲值卡 1000 元
- 獲得 100 元贈送
- 消費 250 元

#### 所需科目
- A-儲值卡（資產科目，初值 0）

#### 記帳步驟

**步驟1：購卡交易**
```
交易類型：轉帳
轉換到：A-儲值卡
從科目：L-信用卡
金額：1000
明細：購買餐廳儲值卡
```

**步驟2：記錄贈送收入**
```
交易類型：收入
進帳從：I-其他收入
到科目：A-儲值卡
金額：100
明細：儲值卡贈送金額
```

**步驟3：記錄消費支出**
```
交易類型：支出
支付給：E-餐飲費
從科目：A-儲值卡
金額：250
明細：使用儲值卡消費
```

**步驟4：結果**
- A-儲值卡餘額：1000 + 100 - 250 = **850 元**

### 狀況 B：含折扣優惠的儲值卡

#### 情境
- 購卡 1000 元
- 享九折優惠
- 消費 250 元實際扣 225 元

#### 記帳步驟

**步驟1：購卡**
```
轉換到：A-儲值卡
從科目：L-信用卡
金額：1000
明細：購買儲值卡
```

**步驟2：折扣消費**
```
支付給：E-餐飲費
從科目：A-儲值卡
金額：225
明細：消費250元打9折（實扣225）
```

**步驟3：結果**
- A-儲值卡餘額：1000 - 225 = **775 元**
- 實際消費：250 元
- 折扣優惠：25 元

#### 說明
- 金額欄填入實際扣款金額（225元）
- 不是消費金額（250元）
- 折扣直接反映在儲值卡餘額

## 6.11 房貸

### 科目設定
- 科目名稱：L-房貸
- 科目類型：負債
- 初始值：當前房貸未還餘額

### 利息記錄

#### 每月產生利息
```
交易類型：支出
支付給：E-利息支出
從科目：L-房貸
金額：8000
明細：房貸利息
```

#### 帳務影響
- 房貸負債增加 8000 元
- 利息支出增加 8000 元

### 還款記錄

#### 償還本金和利息
```
交易類型：轉帳
轉換到：L-房貸
從科目：A-銀行存款
金額：20000
明細：房貸還款
```

#### 帳務影響
- 銀行存款減少 20000 元
- 房貸負債減少 20000 元

### 業務邏輯

#### 每月流程
```
1. 月初產生利息：L-房貸增加 8000（支出）
2. 月中還款：L-房貸減少 20000（轉帳）
3. 實際減少負債：20000 - 8000 = 12000（本金償還）
```

#### 範例：完整一個月
```
期初房貸餘額：1,000,000

1/1  產生利息：
     E-利息支出 → L-房貸 8,000
     房貸餘額：1,008,000

1/5  還款：
     A-銀行存款 → L-房貸 20,000
     房貸餘額：988,000

實際減少本金：1,000,000 - 988,000 = 12,000
```

## 6.12 保險費

### 儲蓄險的記帳方式

#### 分類
- 儲蓄險歸類為**資產項目**
- 因為到期可領回本金或更多

#### 建立科目
- A-XX保險（資產科目）

#### 繳保費時
```
交易類型：轉帳
轉換到：A-XX保險
從科目：A-現金
金額：20000
明細：年度儲蓄險保費
```

#### 保險到期領回時
```
交易類型：轉帳
轉換到：A-現金
從科目：A-XX保險
金額：250000
明細：儲蓄險到期領回
```

#### 如有收益
```
範例：
繳費總計：200,000（10年×20,000）
領回金額：250,000
收益：50,000

收益會反映在資產增加上
```

### 意外險的記帳方式

#### 分類
- 意外險直接作為**支出項目**處理
- 因為無本金返還

#### 繳保費時
```
交易類型：支出
支付給：E-保險
從科目：A-現金
金額：5000
明細：年度意外險保費
```

#### 帳務影響
- 現金減少 5000 元
- 保險支出增加 5000 元
- 無資產殘留

### 核心差異

| 類型 | 會計處理 | 科目類型 | 到期處理 |
|------|---------|---------|---------|
| 儲蓄險 | 資產轉移 | A-XX保險 | 領回本金 |
| 意外險 | 純支出 | E-保險 | 無領回 |

## 6.13 外幣記帳

### 基本原則
- 軟體每個帳本只能處理**單一幣別**
- 建議把外幣部分另建帳本分開記錄
- 例如：台幣帳本、美金帳本、日幣帳本

### 操作範例

#### 情境
- 以台幣 3000 元購買美金 100 元

#### 台幣帳本記錄
```
交易類型：支出
支付給：E-買賣外幣
從科目：A-現金
金額：3000
明細：購買美金100元
```

#### 美金帳本記錄
```
交易類型：收入
進帳從：I-買賣外幣
到科目：A-現金
金額：100
明細：購買美金100元
```

### 完整流程

#### 步驟1：建立外幣帳本
- 帳本名稱：美金帳本
- 幣別：美金（在明細中註明）

#### 步驟2：兌換記錄
**台幣帳本**：
- 記為支出（台幣減少）
- 使用 E-買賣外幣科目

**美金帳本**：
- 記為收入（美金增加）
- 使用 I-買賣外幣科目

#### 步驟3：外幣消費
在美金帳本中正常記帳：
```
支付給：E-餐飲費
從科目：A-現金
金額：20
明細：美國用餐
```

#### 步驟4：兌換回台幣
**美金帳本**：
```
支付給：E-買賣外幣
從科目：A-現金
金額：100
明細：兌換回台幣
```

**台幣帳本**：
```
進帳從：I-買賣外幣
到科目：A-現金
金額：3100
明細：美金兌回（賺100元）
```

### 匯率損益

#### 匯兌收益範例
```
購買：台幣3000 → 美金100
賣出：美金100 → 台幣3100
收益：100元台幣

台幣帳本會自動反映收益
```

#### 匯兌損失範例
```
購買：台幣3000 → 美金100
賣出：美金100 → 台幣2900
損失：100元台幣

台幣帳本會自動反映損失
```

### 關鍵特點
- 外幣交易通過「買賣外幣」科目連接不同幣別帳本
- 收支記錄完成匯兌操作
- 匯率變動自動反映在最終金額
